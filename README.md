# React + TypeScript + Vite — Readme בעברית

תיאור:
- מטרה: אפליקציית Helpdesk ב־React + TypeScript + Vite להצגת ניהול וטיפול בטיקטים עם הרשמה, התחברות ותפקודים לפי תפקיד (Customer, Agent, Admin).
- Backend צפוי: שרת API נפרד (דוגמה בספריה `helpdesk-api-main`) המאזין בברירת מחדל על `http://localhost:4000/`.

תכונות עיקריות:
- Auth: הרשמה (`POST /auth/register`) והתחברות (`POST /auth/login`) — קבלת JWT ושמירתו בצד הלקוח.
- שמירת סשן: JWT נשמר (למשל ב־`localStorage`) ומשמש להגנה על מסלולים.
- דשבורד: דשבורד מותאם לפי תפקיד: Customer / Agent / Admin.
- רשימת טיקטים: `GET /tickets` — תצוגת רשימת טיקטים עם שדות לפחות: Subject, Status (שם), Priority (שם), By, Created (תאריך), id.
- פרטי טיקט + תגובות: דף טיקט עם תגובות ויכולת הוספת תגובות.
- יצירת טיקט: טופס `POST /tickets` עם שדות חובה (Subject, Description, Priority) + ולידציה צד לקוח.
- עדכון סטטוס / הקצאה: `PATCH /tickets/:id` — שינוי סטטוס והקצאת טיקטים (לפי זכויות).
- חוויית משתמש: טקסט/Spinner בזמן טעינה, הודעות שגיאה קריאות (לא רק console.log), הודעות הצלחה ברורות.

תפקידים והתנהגות UI:
- Customer:
  - רואה רק טיקטים שהוא יצר.
  - יכול לפתוח טיקט חדש.
  - יכול להוסיף תגובות לטיקטים שלו.
- Agent (role = `to_assigned`):
  - רואה טיקטים שמוקצים אליו בלבד.
  - יכול לשנות סטטוס לטיקטים שמוקצים לו (`PATCH /tickets/:id`).
  - יכול להוסיף תגובות.
- Admin:
  - רואה את כל הטיקטים.
  - יכול לסנן ולחפש טיקטים.
  - יכול לשנות סטטוס ולהקצות טיקט לסוכן (`PATCH /tickets/:id`).
  - (אופציונלי) מסכי ניהול משתמשים, עדיפות וסטטוסים.

מסכים / Routes מינימליים (חובה):
- `/login`: דף התחברות (Email + Password). שליחה ל־`POST /auth/login`.
- `/register`: דף רישום למשתמשי Customer (שליחה ל־`POST /auth/register`).
- `/dashboard`: דשבורד כללי שמפנה למסכים משניים לפי תפקיד.
- `/tickets`: רשימת טיקטים. (חובה לשלב סינון/חיפוש).
- `/tickets/:id`: פרטי טיקט + תגובות + כפתור להוספת תגובה.
- `/tickets/new`: יצירת טיקט חדש (רק ל־Customer).
- כל הדפים האלה (חוץ מ־`/login` ו־`/register`) מוגנים — דורשים token (Protected routes).

וולידציה וחוויית משתמש:
- שדות חובה: הודעת שגיאה ברורה ליד השדות החסרים.
- טעינה: הצגת Spinner או טקסט בזמן בקשות.
- שגיאות רשת/שרת: הודעה ידידותית למשתמש (לא רק console.log).
- מסכים ריקים: יש הודעה כמו "אין טיקטים להצגה" במקום מסך ריק.

קבצי קונפיגורציה חשובים:
- `src/config/api.ts`: כתובת בסיס ל־API. ברירת מחדל בפרויקט: `http://localhost:4000/`. עדכן אם ה־backend רץ במקום אחר.
